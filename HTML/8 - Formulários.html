Form Validations

Serverside validation: the data sent in the form is analysed by the server to see if it is in a acceptable pattern (ex: phone numbers must be 10 digits, which are all numbers)

Browser validation: the browser checks the data format before the form is even submited, as the user fills the form (ex: an error message that goes "the password must be at least 8 digits long")

Atributes:

Required: won't let the user submit the form uless they filled the input
Syntax: <input type=" " ... required>

Input length: constrols how big/small the user can make the data
Text length: minlength="" and maxlength=""
Numeric length: min="" and max=""
Syntax: <input type=" " ... minlength=" " maxlength=" ">
Pattern: the type of input sets the expected format for the data (ex: if type="email", the expected input must have a "@")
